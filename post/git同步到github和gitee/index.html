git同步到github和gitee

Posted by Mr.Be1ieVe on Monday, January 13, 2020

==按照此说明做下来保证可用，但我也还没完全研究透彻所以有诸多不足请多原谅==

一、准备工作

1、建立一个仓库

https://gitee.com/

2、下载git

https://git-scm.com/ 可以跟随这个 https://blog.csdn.net/sishen47k/article/details/80211002 安装

二、开始操作

1、创建SSHkey

打开Git Bash，输入ssh-keygen -t rsa 会分别提示你文件名，密码（可不设置），确认密码。一般三次回车就可以 在 C:\Users\“你的用户名”.ssh 目录下就会有.pub结尾的文件，用记事本打开，复制里面的东西，这就是你的SSH key

2、添加key到Gitee

gitee->设置->SSH公钥 把key粘贴进去，标题默认是本机的名字

3、git连接到gitee

https连接

拷贝下来你仓库的链接，如 https://gitee.com/xxxxx 然后在你想同步的文件夹上右键 点击GIt Bash Here，输入

git init
git remote add origin https://gitee.com/xxxxx

add origin的origin是给他的一个命名 然后设置config，分别设置名字和邮箱

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

ssh链接 ==推荐==

$ssh -T git@gitee.com
Hi XXX! You've successfully authenticated, but GITEE.COM does not provide shell access.

则代表ssh链接成功

git remote add gitee git@gitee.com:YourUserName/YourProjectName.git

YourUserName和YourProjectName都是要具体看你自己的实际用户名和仓库名字 即可，可以通过git remote -v查看

添加你的文件git add .

git commit -m "备注"

推送 git push origin/gitee master 拉取 git pull origin/gitee master ==使用origin或者gitee取决于你给他的命名==

==这里的master应该也可以换成develop分支上传==

深入理解github模式请戳 https://blog.csdn.net/weixin_33547926/article/details/78545042

3、再次更新之后如何同步

git status
git add -A
git commit -a -m "备注"
git push origin/gitee master -f

4、取消暂存

git status
git reset filename

使用VScode更新

==先保存文件，再暂存更改，随后拉取（pull），即可提交。提交完后再推送到（push）==
